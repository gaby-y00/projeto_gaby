# Definição do Projeto que será desenvolvido em Django 
Aluna: Gabrielly Silveira Barros 
 
# 1. Qual o nome do seu sistema? 
Meu projeto irá se chamar “Agenda Fluida”  
 
# 2. Qual projeto você vai desenvolver? 
Vou desenvolver um sistema web para organizar rotinas pessoais, separando 
tarefas por categorias, com níveis de prioridade.  
 
# 3. Qual é o objetivo do seu sistema? 
O sistema pretende facilitar o gerenciamento diário de compromissos, 
estudos, finanças e tarefas pessoais, de forma simples e organizada.  
 
# 4. Para quem ele é feito (público-alvo)? 
Estudantes, profissionais e qualquer pessoa que precise organizar sua rotina 
de maneira prática e visualmente agradável.  
 
# 5. Qual problema real seu sistema pretende resolver? 
A dificuldade de manter uma rotina organizada devido ao excesso de 
aplicativos complexos, que acabam desmotivando o uso no dia a dia.  
 
# 6. Quais serão as principais funções do sistema? 
As principais funções serão: cadastro de tarefas, categorização por áreas 
(estudo, finanças, pessoal), níveis de prioridade, e anotações livres.  
 
# 7. Quantos tipos de usuários diferentes o sistema terá? 
Usuário padrão. 
 
# 8. Quais telas/interfaces você pretende que o sistema tenha? 
Tela de login, tela de cadastro, dashboard (visão geral das tarefas), tela de 
criação de tarefas, lista de tarefas do dia. 
 
# 9. Há algum motivo pessoal, acadêmico ou profissional que te incentivou a desenvolver esse sistema? 
Estou a procura de um sistema que me ajude na organização das minhas 
tarefas, porém, depois de testar vários e não me identificar com nenhum, 
decidi trazer essa questão para o meu projeto. 
  
# 10.  Quais informações (clsdados) o sistema precisará armazenar? 
O sistema precisará armazenar dados dos usuários (como nome, e-mail e 
senha) e informações relacionadas às tarefas, como título, descrição 
(anotações), categoria (estudo, finanças ou pessoal), data (prazo ou dia da 
atividade), nível de prioridade (urgente, médio ou flexível) e tipo da tarefa (fixa 
ou temporária).  

# 11. O sistema será utilizado em computador, celular ou ambos? 
Ambos. 

# 12. Você pretende usar alguma tecnologia específica além do que foi visto em aula? 
Não. 

# 13. Quais são as principais dificuldades que você imagina encontrar no 
desenvolvimento? 
Organizar corretamente os dados das tarefas (como categorias, prioridades e 
tipos), implementar o sistema de login de usuários e estruturar as 
funcionalidades de forma simples e intuitiva. Além disso, pode haver 
dificuldade em manter o sistema organizado sem adicionar complexidade 
excessiva, já que o objetivo é ser uma ferramenta fluida.
